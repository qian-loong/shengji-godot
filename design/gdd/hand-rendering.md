# P2 手牌渲染

> **Status**: Designed
> **Author**: user + game-designer
> **Last Updated**: 2026-04-06
> **Implements Pillar**: 支柱 3（对局自足）

## Overview

**手牌渲染系统**负责玩家手牌的视觉呈现：牌面显示、排序布局、选中交互、花色与点数的清晰辨识。本系统是玩家与游戏最密切的交互界面——手牌的可读性直接影响决策效率。

## Player Fantasy

玩家需要在手牌中快速找到想出的牌。花色必须一眼可分辨（写实风格，颜色对比鲜明），主副牌分区明确，排序符合直觉。本系统服务的情感是**清晰感**——"我能立刻看清手里有什么"。

## Detailed Design

### Core Rules

#### 1. 牌面显示

每张牌显示：
- 花色图标（♠♥♦♣）+ 颜色（黑/红）
- 点数文字（2-10, J, Q, K, A）
- 王牌特殊显示（大王/小王，红/黑配色）
- 写实风格，四种花色在移动设备小屏上清晰可辨

#### 2. 手牌排序

调用 C1 的 `get_sort_value()` 排序，布局规则：
- **主牌在左，副牌在右**（或可配置为主牌在右）
- 主牌内部按 L1→L5 层级排序（普通主花色牌 → 非主花级牌 → 主花级牌 → 小王 → 大王）
- 副牌内部按花色分组，每组内按点数排序
- 花色间用间距分隔，视觉上分组明确

#### 3. 选中交互

- 点击牌 → 牌上浮（表示选中）
- 再次点击 → 取消选中（牌回落）
- 支持多选（甩牌、拖拉机需要选多张）
- 选中牌高亮/上浮后，点击"出牌"按钮确认
- 不合法的出牌组合：出牌按钮灰掉或提示原因

#### 4. 发牌阶段的渐进显示

发牌过程中牌逐张出现在手牌区，配合 F2 的发牌事件流：
- 每收到 1 张牌 → 插入手牌并重新排序（带动画）
- trump_suit 确定前：级牌特殊标记（高亮边框），提示"可亮主"
- trump_suit 确定后：手牌按主副重新分区排列

### States and Transitions

| 状态 | 说明 |
|------|------|
| **Empty** | 无手牌（局间） |
| **Dealing** | 发牌中，逐张增加 |
| **Idle** | 等待出牌（非玩家回合） |
| **Selecting** | 玩家选牌中 |
| **Confirmed** | 出牌确认，选中牌飞出 |

### Interactions with Other Systems

| 方向 | 系统 | 接口 |
|------|------|------|
| ← F1 牌型定义 | Card 显示属性（suit、rank、joker_type） |
| ← C1 主副牌判定 | get_sort_value() 排序 |
| ← F2 牌组管理 | 发牌事件流 |
| ← C7 对局状态机 | 当前阶段（控制交互启用/禁用） |
| → C2 出牌合法性校验 | 选中牌送 C2 校验 |

## Formulas

手牌布局公式：
- 牌宽 = 屏幕宽度 / max(手牌数 × 0.4, 10)（重叠显示，确保都能看到）
- 重叠率随手牌数增加而增大（25 张牌约 60% 重叠）

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|---------|---------|
| E1 | **33 张手牌（配底阶段）** | 增大重叠率，确保所有牌可见 |
| E2 | **trump_suit 变更（反主）** | 手牌重新排序，带过渡动画 |
| E3 | **横屏/竖屏切换** | 手牌区自适应布局 |

## Tuning Knobs

| 参数 | 类型 | 默认值 | 影响 |
|------|------|--------|------|
| `trump_side` | enum | Left | 主牌显示在左侧还是右侧 |
| `card_scale` | float | 1.0 | 牌面缩放（适配不同屏幕） |

## Acceptance Criteria

| # | 测试条件 | 预期结果 |
|---|---------|---------|
| AC1 | 25 张手牌全部可见 | 无牌被完全遮挡 |
| AC2 | 主牌和副牌视觉分组 | 有明显间距分隔 |
| AC3 | 四种花色可辨 | 在手机屏幕上 ♠♥♦♣ 一眼区分 |
| AC4 | 选中牌上浮 | 视觉反馈即时 |
| AC5 | 发牌逐张出现 | 每张牌有插入动画 |
