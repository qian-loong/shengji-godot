# P4 亮主/抢主 UI

> **Status**: Designed
> **Author**: user + game-designer
> **Last Updated**: 2026-04-06
> **Implements Pillar**: 支柱 2（规则即策略）

## Overview

**亮主/抢主 UI**提供发牌阶段和反主阶段的亮主交互界面。玩家通过此界面选择是否亮主、亮什么花色，并能看到其他玩家的亮主声明。

## Player Fantasy

亮主是紧张的决策时刻——"要不要亮？什么时候亮？"。UI 需要让玩家快速操作（发牌不等人），同时清晰展示当前亮主状态。本系统服务的情感是**决策紧迫感**。

## Detailed Design

### Core Rules

#### 1. 亮主按钮

发牌阶段，当玩家手中有合法声明条件时：
- 显示亮主选择器（可选花色列表，基于手中级牌/王牌）
- 无合法条件时选择器不出现
- 玩家点击花色 → 发出亮主声明给 C3

#### 2. 亮主状态展示

- 当前谁亮了主 → 该座位高亮 + 花色图标
- 首局抢主模式 → 显示"先到先得"提示
- 非首局 → 庄家标记 + "等待定主"

#### 3. 反主窗口 UI

配底完成后进入反主窗口时：
- 如果玩家（攻方）有反主条件 → 显示"反主"按钮
- 显示当前庄家的亮主强度（供玩家判断能否反）
- 玩家选择反主或放弃
- AI 反主时 → 显示反主动画和通知

### States and Transitions

| 状态 | 说明 |
|------|------|
| **Hidden** | 非亮主阶段，UI 隐藏 |
| **BidAvailable** | 玩家可亮主，显示选择器 |
| **BidLocked** | 已有人亮主（首局不可覆盖） |
| **CounterAvailable** | 反主窗口，玩家可反主 |
| **CounterLocked** | 反主完成或窗口关闭 |

### Interactions with Other Systems

| 方向 | 系统 | 接口 |
|------|------|------|
| ← C3 亮主/抢主/反主 | 当前亮主状态、可用声明列表 |
| ← C7 对局状态机 | 当前阶段（Bidding / CounterWindow） |
| → C3 | 玩家的亮主/反主声明 |

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|---------|---------|
| E1 | **发牌速度快于玩家操作** | 亮主选择器持续显示直到发牌结束，不因发牌完成而消失 |
| E2 | **AI 亮主** | 显示亮主通知动画（AI 座位弹出花色图标） |
