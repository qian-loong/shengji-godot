# P3 出牌/动画反馈

> **Status**: Designed
> **Author**: user + game-designer
> **Last Updated**: 2026-04-06
> **Implements Pillar**: 支柱 3（对局自足）

## Overview

**出牌/动画反馈系统**负责出牌过程的视觉表现：牌从手牌飞入出牌区、跟牌依次出现、赢墩牌堆收回、甩牌失败动画等。本系统不参与逻辑判定，只消费 C2/C7 的结果进行视觉呈现。

## Player Fantasy

出牌时的视觉反馈需要干脆利落——牌飞出、落定、赢墩收回，节奏明快。过慢则拖沓，过快则看不清。本系统服务的情感是**节奏感**。

## Detailed Design

### Core Rules

#### 1. 动画类型

| 动画 | 触发条件 | 表现 |
|------|---------|------|
| 出牌 | 玩家/AI 确认出牌 | 牌从手牌区飞入出牌区对应位置 |
| 跟牌 | 跟牌方出牌 | 同上，按座位顺序依次 |
| 赢墩收回 | determine_winner 返回结果 | 短暂展示（1-2秒），然后牌堆飞向赢家 |
| 甩牌失败 | C2 返回甩牌失败 | 选中牌展开 → 失败标记 → 最小单牌飞出，其余回收 |
| 主牌杀 | 跟牌出主牌 | 主牌出牌时有额外视觉强调（边框发光） |
| 得分提示 | 该墩有分牌且被赢 | 分值浮动数字（"+25"等） |

#### 2. 动画节奏

| 动作 | 时长 |
|------|------|
| 单张出牌飞行 | 200ms |
| 跟牌间隔 | 300ms |
| 赢墩展示 | 1000ms |
| 赢墩收回 | 300ms |
| 得分浮动 | 800ms |

所有时长可由 `animation_speed` 倍率调整。

#### 3. 非玩家回合的自动播放

AI 出牌时自动播放动画。玩家回合等待玩家操作后播放。

### States and Transitions

P3 是事件驱动的——监听 C7 的出牌事件，播放对应动画。无持续状态。

### Interactions with Other Systems

| 方向 | 系统 | 接口 |
|------|------|------|
| ← C7 对局状态机 | 出牌事件、赢墩事件 |
| ← C2 出牌合法性校验 | 甩牌失败事件 |
| ← C5 分值追踪 | 墩分值（得分提示） |
| ← P2 手牌渲染 | 牌的起始位置（飞行起点） |
| 嵌入 P1 | 出牌区空间 |

## Formulas

无。

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|---------|---------|
| E1 | **快速模式（跳过动画）** | animation_speed 设为极大值或直接跳过 |
| E2 | **甩牌多张同时飞出** | 多张牌同时飞入但有微小偏移（扇形展开） |
| E3 | **最后一墩** | 与普通墩相同，无特殊动画（结果由 P5 结算界面展示） |

## Dependencies

| 依赖方向 | 系统 | 类型 |
|---------|------|------|
| ← C7, C2, C5 | 硬依赖 |
| ← P2 | 软依赖 |
| 嵌入 P1 | 空间定位 |

## Tuning Knobs

| 参数 | 类型 | 默认值 | 影响 |
|------|------|--------|------|
| `animation_speed` | float | 1.0 | 动画速度倍率（2.0 = 双倍速） |

## Acceptance Criteria

| # | 测试条件 | 预期结果 |
|---|---------|---------|
| AC1 | 玩家出牌 | 牌飞入出牌区，手牌减少 |
| AC2 | 4 人出牌完毕 | 赢墩者标记，牌堆收回 |
| AC3 | 甩牌失败 | 失败动画 + 最小单牌飞出 |
| AC4 | 墩中有分牌被攻方赢 | 显示"+N"浮动数字 |
| AC5 | animation_speed = 2.0 | 所有动画加速一倍 |
