# P5 结算界面

> **Status**: Designed
> **Author**: user + game-designer
> **Last Updated**: 2026-04-06
> **Implements Pillar**: 支柱 3（对局自足）

## Overview

**结算界面**在每局结束时展示完整的结算结果：攻方得分、抠底详情、升级结果、下一局信息。本系统消费 C6 的结算数据进行视觉呈现。

## Player Fantasy

结算是"揭晓答案"的时刻——特别是抠底翻盘时的戏剧性。结算界面需要有节奏地展示信息，让玩家感受到结果的重量。本系统服务的情感是**揭晓感**。

## Detailed Design

### Core Rules

#### 1. 结算展示流程（有节奏，非一次性）

| 步骤 | 内容 | 时长 |
|------|------|------|
| 1 | 出牌阶段攻方得分 | 显示数字 |
| 2 | 最后一墩赢家（攻方/庄家方） | 高亮标记 |
| 3 | 底牌翻开展示 | 逐张翻开动画 |
| 4 | 底牌分值 × 抠底倍数 | 数字滚动（如 "15 × 4 = 60"） |
| 5 | 最终得分 | 大字展示 |
| 6 | 升级结果 | "攻方升 2 级" / "庄家方守住" |
| 7 | 下一局信息 | 新级牌、新庄家 |

如果庄家方赢最后一墩（无抠底），跳过步骤 3-4。

#### 2. 游戏结束展示

当 C6 判定游戏结束时：
- 替换步骤 7 为"游戏结束！[队伍名] 获胜！"
- 显示整场战绩摘要（总局数、各方最高级等）

#### 3. 操作按钮

- "下一局" → 触发 C7 进入 InRound
- "返回大厅" → 触发 C7 进入 Lobby
- 游戏结束时："再来一局" / "返回大厅"

### States and Transitions

| 状态 | 说明 |
|------|------|
| **Hidden** | 非结算阶段 |
| **Animating** | 结算信息分步展示中 |
| **Complete** | 所有信息展示完毕，等待玩家操作 |

### Interactions with Other Systems

| 方向 | 系统 | 接口 |
|------|------|------|
| ← C6 升级结算 | 结算数据（得分、倍数、升级结果等） |
| ← C4 抠底/配底 | 底牌内容（翻开展示） |
| ← C7 对局状态机 | Settlement 阶段触发显示 |
| → C7 对局状态机 | 玩家点击"下一局"或"返回" |

## Formulas

无。

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|---------|---------|
| E1 | **底牌无分牌** | 步骤 4 显示 "0 × N = 0"，快速跳过 |
| E2 | **攻方 0 分** | 特殊展示："庄家大胜！攻方颗粒无收" |
| E3 | **攻方 200 分** | 特殊展示："攻方全收！" |
| E4 | **玩家快速点击跳过** | 允许点击屏幕跳过动画，直接显示最终结果 |

## Dependencies

| 依赖方向 | 系统 | 类型 |
|---------|------|------|
| ← C6, C4, C7 | 硬依赖 |
| 嵌入 P1 | 全屏覆盖或弹窗 |

## Tuning Knobs

| 参数 | 类型 | 默认值 | 影响 |
|------|------|--------|------|
| `settlement_animation` | bool | true | 是否播放分步动画（false 则直接展示结果） |
