# P1 桌面布局 UI

> **Status**: Designed
> **Author**: user + game-designer
> **Last Updated**: 2026-04-06
> **Implements Pillar**: 支柱 3（对局自足）

## Overview

**桌面布局系统**定义 4 人牌桌的空间结构：玩家位置、出牌区、信息显示区域的屏幕布局。本系统是所有其他 UI 系统的空间容器。

## Player Fantasy

玩家应感觉自己坐在一张真实的牌桌前——搭档在对面，对手在两侧，出牌区在中间。布局符合物理直觉，无需学习。本系统服务的情感是**沉浸感**。

## Detailed Design

### Core Rules

#### 1. 座位布局

```
        北(2) — AI 搭档
西(3)                 东(1)
  AI 对手           AI 对手
        南(0) — 玩家
```

- 玩家固定在屏幕下方（南）
- 搭档在对面（北）
- 对手在左右（东/西）
- 座位编号按逆时针递增：南(0)→东(1)→北(2)→西(3)

#### 2. 屏幕区域划分

| 区域 | 位置 | 内容 |
|------|------|------|
| 玩家手牌区 | 屏幕底部 | P2 手牌渲染 |
| 出牌区 | 屏幕中央 | 本墩 4 人出的牌 |
| AI 手牌区（×3） | 上/左/右 | AI 手牌背面（显示张数） |
| 信息栏 | 屏幕顶部或侧边 | 当前级、主花色、得分、庄家标记 |
| 操作区 | 手牌区上方 | 出牌/取消按钮 |

#### 3. 信息栏内容

- 当前级牌（current_rank）
- 主花色图标（trump_suit，公主局显示"无主"）
- 攻方当前得分 / 升级阈值
- 庄家标记（哪个座位是庄家）
- 剩余手牌数（每个 AI 的手牌张数）

### States and Transitions

桌面布局在对局中保持不变。仅在亮主阶段 trump_suit 确定时更新信息栏。

### Interactions with Other Systems

| 方向 | 系统 | 接口 |
|------|------|------|
| ← C7 对局状态机 | 当前阶段（控制区域显示/隐藏） |
| ← C5 分值追踪 | 实时得分 |
| ← C3 亮主/抢主/反主 | trump_suit、庄家座位 |
| ← F3 规则配置系统 | current_rank |
| 包含 P2 | 手牌渲染嵌入手牌区 |
| 包含 P3 | 出牌动画在出牌区播放 |

## Formulas

无。布局由屏幕尺寸和固定比例决定。

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|---------|---------|
| E1 | **极小屏幕（<5 寸）** | 最小字号限制，信息栏可折叠 |
| E2 | **横竖屏切换** | 布局自适应重排 |
