# 游戏概念文档 — 双升对局

> 版本：v0.1 | 阶段：概念 | 最后更新：2025-01

---

## 一句话定位

一款以搭档配合为核心体验、高可配置规则为底座的竞技型单机双升卡牌游戏，目标平台 Android，1 人 + 3 AI，随开随打。

---

## 创意简报

开发者是有扎实 C++/Qt 背景的程序员，正在做第一个游戏项目。真正的驱动力不是「做一款卡牌游戏」，而是「把熟悉但市面上常见实现不准确的双升规则正确数字化」。最看重的乐趣是规则系统本身带来的研究感、设计感与代码架构的优雅感。产品优先级：先高保真还原家乡玩法，再通过高可配置规则兼容更多地区与玩家习惯。

---

## 核心定位

**方向**：《升牌对局》—— 聚焦真实牌桌体验的单机双升，内置高质量 AI 与多套规则配置，让玩家能快速进入熟悉的对局。

**核心动词**：出牌

**核心幻想**：「我随时都能开一桌像真人一样的双升。」

**主要 MDA 美学**：Challenge（挑战）/ Fellowship（搭档模拟）

---

## 三大设计支柱

### 支柱 1：配合可感知
AI 队友的行为必须让玩家感受到「它在配合我」，而不是「它在独立打牌」。搭档配合是本游戏最核心的情感爽点，AI 的配合意图必须在出牌决策中可见、可感知。

### 支柱 2：规则即策略
每一条规则配置都应对博弈策略产生可见影响。规则不是皮肤或外观选项，而是玩法结构的组成部分。改变规则 = 改变策略空间。

### 支柱 3：对局自足
一局 10–15 分钟，随开随打，不依赖外部进度系统（无强制升级树、无每日任务）。所有成就感来自对局本身的决策质量与博弈深度。

---

## 核心循环

```
进入对局
  → 发牌 / 亮主（或抢主/固定主/无主）
  → 抠底 / 配底
  → 出牌阶段（循环）
      ↳ 玩家读牌、判断主副、决策出手
      ↳ AI 队友体现配合意图（传递信息、让路、接管）
      ↳ AI 对手体现压制与博弈（封主、截牌、算分）
  → 一手结束：算分 / 记录已出牌
  → 全局结束：升级结算（攻守方计算分差，决定升几级）
  → 复盘冲动：「这局哪里打错了？」→ 再来一局
```

**次级循环（规则配置层）**：
```
调整规则参数 → 进入对局 → 感受规则对节奏/策略的影响 → 理解规则如何塑造博弈 → 进一步调整
```

## 可配置规则系统

以下规则项均可在开局前配置：

| 规则项 | 说明 |
|--------|------|
| 亮主 / 抢主 / 反主 / 固定主 / 无主 | 主牌产生方式 |
| 副牌数量 | 支持 1–2 副牌 |
| 级牌是否算主 | 级牌归属花色的判定 |
| 大小王是否永远为主 | 王牌在无主时的处理 |
| 是否允许甩牌 | 甩牌规则开关 |
| 跟牌是否必须严格匹配结构 | 跟牌约束强度 |
| 拖拉机是否允许级牌或混合主牌成分 | 拖拉机构成规则 |
| 抠底与升级规则计算方式 | 底牌分值倍数与升级计算 |

## 目标玩家画像

- 熟悉双升规则、有地方玩法习惯的玩家
- 在意规则细节、愿意研究博弈的「规则玩家」
- 想随时开一桌但身边没有真人对局环境的玩家

**粘性来源**：规则的正确性与 AI 质量，而非外部进度系统。

## 技术概览

| 维度 | 决策 | 说明 |
|------|------|------|
| 引擎 | Godot 4 | 轻量免费，2D UI 成熟，安卓导出支持好，内置网络模块 |
| 平台 | Android（优先） | 手机竖屏/横屏适配 |
| 语言 | GDScript | 类 Python 语法，一人项目上手快 |
| AI 架构 | 规则引擎 + 决策树（MVP）→ 推理 AI（迭代） | MVP 先做规则正确，再做配合意图 |
| 联机 | 预留设计，MVP 不实现 | Godot 内置 ENet/WebSocket |

---

## 规模与里程碑规划（草案）

**MVP 目标**：可完整打完一局双升，规则可配置，AI 行为规则正确。

| 里程碑 | 内容 |
|--------|------|
| M0 — 规则引擎 | 牌型定义、出牌合法性校验、升级计算，无 UI |
| M1 — 基础对局 | 发牌、亮主、出牌、结算，简单 AI |
| M2 — 规则配置 | 所有规则项可配置，UI 完整 |
| M3 — AI 提升 | AI 配合意图、难度梯度 |
| M4 — 安卓发布 | 导出、适配、性能优化 |
| M5 — 联机（未来） | 多人联机支持 |

---

## 风险登记

| 风险 | 级别 | 缓解策略 |
|------|------|----------|
| AI 配合意图难以实现 | 高 | MVP 只做规则正确 AI，配合意图在 M3 迭代 |
| 规则系统复杂度超出预期 | 中 | 先实现家乡默认规则，配置项逐步解锁 |
| Godot 学习曲线 | 中 | GDScript 对 C++ 开发者友好，有大量文档 |
| 安卓 UI 适配 | 低 | Godot 4 有成熟的屏幕适配方案 |
