# Sprint 3 — M1 图形 UI 表现层

> **Status**: In Progress
> **Created**: 2026-07-01
> **Sprint Goal**: 将 TUI 终端原型逐步替换为 Godot 图形界面，实现横屏牌桌视觉、程序化卡牌渲染、选牌交互、亮主界面与结算画面。保留 TUI 可用。

---

## Sprint Info

- **周期**：3–4 周（弹性节奏）
- **屏幕方向**：横屏优先（1280×720 设计基准，Android landscape）
- **美术策略**：程序化占位符（纯代码绘制花色+点数），后续替换正式美术资源
- **开发策略**：增量替换——保留 `tui_game.tscn` 可用，新建 `gui_game.tscn` 图形场景

## 架构设计

```
gui_game.tscn (主场景)
├── Background         — 牌桌背景（深绿色）
├── InfoBar            — 顶部信息栏（级、主、庄家、得分）
├── TableCenter        — 中央出牌区（4 个 CardView 位置）
├── SeatArea × 4       — 四方座位区（头像/名字/手牌数）
├── HandArea           — 底部玩家手牌（CardView 扇形排列）
├── BidPanel           — 亮主浮层（P4）
├── ActionBar          — 操作按钮区（出牌/取消）
└── SettlementOverlay  — 结算覆盖层（P5）
```

核心组件：

- `CardView` (Control) — 程序化绘制单张卡牌，支持选中、高亮、翻面
- `HandDisplay` (Control) — 管理手牌水平排列、选牌交互
- `GuiGame` (Control) — GUI 宿主，消费 SessionController API
